🎯 Общая характеристика проекта

| Атрибут                    | Значение                                                                                                                                                                                                                                           |
|----------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Тип                        | Программный проект (учебный)                                                                                                                                                                                                                       |
| Максимальная оценка        | 8,2 балла (учебный проект)                                                                                                                                                                                                                         |
| Описание                   | Исследование: адаптивное кодирование видео на основе Region of Interest (ROI). Продукт: система кодирования видео, которая перераспределяет качество внутри кадра (битрейт/квантование/маски качества) в зависимости от значимости областей (ROI). |
| Цель продукта              | Разработка системы, позволяющей повышать визуальное качество важных областей видео (например, лица, объекты, зона взгляда) при ограниченном битрейте за счёт деградации незначимых областей.                                                       |
| Целевая аудитория          | Системы видеонаблюдения, VR/AR, видеоконференции, облачные игровые сервисы, edge-решения, сервисы обработки и сжатия видео.                                                                                                                        |
| Технологии (рекомендуемые) | FFmpeg, OpenCV, модели saliency/gaze/object detection, Go/C++, AV1/HEVC/VVC (ROI, tiles, QP map), Docker                                                                                                                                           |

## 📌 Этап 2: Представление проекта

Дата: 31.01.2026

Формат: Презентация

Документы: Слайды, ТЗ

### 🔹 Требования к защите

* Чётко сформулированы:
    * Проблема: ограниченный битрейт → равномерная деградация качества → потеря важной информации.
    * Решение: адаптивное распределение качества внутри кадра (ROI-based encoding).
    * Целевая аудитория.
* Описана архитектура:
    * генерация ROI,
    * маппинг качества,
    * кодирование,
    * сохранение результата и отчетов.
* Представлен план реализации.

| Критерий     | Вес | Описание                                                          |
|--------------|-----|-------------------------------------------------------------------|
| Продукт      | 25% | Видение системы: как работает ROI, где применяется                |
| Польза       | 25% | Улучшение QoE при фиксированном битрейте                          |
| Пользователь | 20% | Обоснование аудитории и сценариев использования                   |
| Технологии   | 20% | ROI detection, encoder control (QP map / tiles) , FFmpeg pipeline |
| Развитие     | 10% | AV1/VVC, gaze tracking, edge computing                            |

## 📌 Этап 3: PoC (Proof of Concept)

Дата: 21.02.2026

Формат: Демонстрация + видео + репозиторий

### 🔹 Требования к защите

* Подтверждена техническая реализуемость:
    * Чтение входного видеофайла.
    * Выделение ROI (например:
        * детекция движения,
        * bounding box объекта,
        * saliency map).
    * Применение различного качества:
        * через QP offset или
        * простое разбиение на области (mask).
* Демонстрация:
    * ROI область с высоким качеством,
    * остальная часть — с пониженным.

| Критерий                  | Вес | Описание                                                       |
|---------------------------|-----|----------------------------------------------------------------|
| Техническая реализуемость | 60% | Подтверждение возможности выделения ROI и управления качеством |
| Демонстрация              | 20% | Видео: обычное кодирование vs ROI                              |
| Код и документация        | 20% | Репозиторий, README                                            |

## 📌 Этап 4: Прототип

Дата: 16.03.2026

Формат: Демо + отчёт + git

### 🔹 Требования к защите

* Реализован прототип:
    * Поддержка нескольких типов ROI:
        * движение,
        * объекты (OpenCV),
        * статическая зона.
