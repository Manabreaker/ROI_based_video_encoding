# 1\. Резюме проекта

Проект разрабатывает систему потоковой передачи видео, которая перераспределяет качество внутри кадра: важные области (Region of Interest, ROI) кодируются лучше, а менее важные области кодируются грубее. Это позволяет при фиксированном канале сохранить детализацию там, где пользователь или алгоритм ожидает значимую информацию: лицо, объект, зона движения, указатель оператора или зона взгляда.

## 1.1. Проблема

При ограниченном битрейте обычное равномерное кодирование ухудшает весь кадр. В результате критически важные детали могут потеряться, хотя большая часть кадра может быть менее значимой для пользователя.

## 1.2. Решение

Решение - адаптивное кодирование на основе ROI: модуль анализа кадра формирует ROI, затем ROI преобразуется в карту качества или маску, а encoder/output pipeline использует её для управления QP, битрейтом или tile-quality selection.

## 1.3. Решение для Этапа 2

Для защиты Этапа 2 фиксируется базовая ветка реализации: single-stream ROI pipeline. На PoC допускается mask-based обработка через FFmpeg, потому что она воспроизводима на локальных видеофайлах и позволяет быстро измерить baseline vs ROI без необходимости сразу реализовывать WebRTC, gaze-tracking и DASH-клиент. Следующий шаг после PoC - заменить mask-based preprocessing на encoder-level QP/ROI map.

# 2\. Термины и сокращения

| **Термин**             | **Определение**                                                                                          |
|------------------------|----------------------------------------------------------------------------------------------------------|
| **ROI**                | Region of Interest - важная область кадра, которую нужно сохранить в более высоком качестве.             |
| **Foveated streaming** | Частный случай ROI, где важная область соответствует зоне взгляда/фовеации.                              |
| **QP**                 | Quantization Parameter - параметр квантования; меньший QP обычно даёт лучшее качество и больший битрейт. |
| **QP offset map**      | Карта смещений QP по блокам кадра, передаваемая энкодеру.                                                |
| **Tiles/SRD**          | Пространственное разбиение кадра и описание spatial relationships в MPEG-DASH для выбора частей видео.   |
| **QoE**                | Quality of Experience - воспринимаемое пользователем качество.                                           |

# 3\. Целевая аудитория и сценарии

| **Сценарий**                      | **Польза**                                                                                                  |
|-----------------------------------|-------------------------------------------------------------------------------------------------------------|
| **Системы видеонаблюдения**       | Лица, номера, движущиеся объекты важнее фона. ROI позволяет экономить канал без потери критических деталей. |
| **Дроны, роботы, edge-камеры**    | Оператор или алгоритм задаёт объект интереса; канал часто ограничен.                                        |
| **Видеоконференции**              | Лица/говорящий важнее фона; можно улучшать воспринимаемое качество при фиксированном bitrate budget.        |
| **Cloud gaming / remote desktop** | Требуются низкая задержка и высокая детализация в зоне внимания.                                            |
| **VR/AR и 360°**                  | Viewport/gaze-driven зоны могут получать лучшее качество; tile/SRD уместны как следующий уровень.           |

# 4\. Архитектура

![](images/8df5c14f-4576-4fc8-8784-63230e74c6a3.png)
Иллюстрация идеи: центр/ROI кодируется качественнее, периферия может быть дешевле по битрейту.

| **Блок**                | **Назначение**                                                                                       |
|-------------------------|------------------------------------------------------------------------------------------------------|
| **1\. Ingest**          | Получение видео: файл, камера или RTSP. Для PoC достаточно файла/локальной камеры.                   |
| **2\. ROI generator**   | Выделение ROI: static box, motion detection, object detection; gaze/saliency как расширение.         |
| **3\. Quality mapper**  | Преобразование ROI в карту качества: QP offsets по блокам или маска для последующей обработки.       |
| **4\. Encoder**         | PoC: FFmpeg mask-based H.264 output. Развитие: SVT-AV1 ROI map, Kvazaar delta QP map или libaom ROI map. |
| **5\. Delivery/output** | PoC: файл. Далее: RTSP/WebRTC или DASH/GPAC в зависимости от этапа.                                  |
| **6\. Evaluation**      | Битрейт, ROI-PSNR/SSIM/VMAF, full-frame метрики, latency/cost при необходимости.                     |

## 4.1. Архитектурное решение

Базовая ветка: single-stream ROI. На PoC - mask-based ROI через FFmpeg; для прототипа - переход к QP/ROI-map там, где это поддерживает энкодер.

## 4.2. Альтернативная ветка

Tile/subpicture delivery через MPEG-DASH SRD/GPAC оставить для развития. Этот путь полезен для 360°/VR и VOD, но требует подготовки tiled-контента, совместимых потоков, MPD/SRD и клиентской логики.

# 7\. Технологический стек

| **Слой**             | **Выбор**                                                             | **Обоснование**                                                                  |
|----------------------|-----------------------------------------------------------------------|----------------------------------------------------------------------------------|
| **ROI detection**    | OpenCV; static bbox; motion detection; object detector как расширение | Лёгкий старт и хорошая демонстрируемость.                                        |

| **Realtime later**   | WebRTC; AV1 RTP payload                                               | Уместно после offline PoC, когда базовая ROI-логика доказана \[S6\]\[S7\].       |

## 7.1. Проверка реализуемости по источникам

SVT-AV1 документирует ROI/QP Offset Map file: строка содержит номер кадра и QP offsets для 64x64 блоков, а пример запуска использует --roi-map-file \[S1\].

Kvazaar поддерживает параметр --roi &lt;filename&gt;, который читает delta QP map из text/bin файла и масштабирует карту под размер видео \[S2\].

GPAC показывает HEVC tile-based adaptation для DASH/SRD как отдельный путь, но это не требуется как минимум PoC \[S3\].

# 9\. Метрики

| **Группа**             | **Метрики**                             | **Зачем**                                        |
|------------------------|-----------------------------------------|--------------------------------------------------|
| **Bitrate**            | avg, p95/p99, размер файла              | Показывает экономию канала.                      |
| **Full-frame quality** | PSNR, SSIM, VMAF                        | Контроль общего ухудшения кадра.                 |
| **ROI quality**        | ROI-PSNR/SSIM/VMAF по маске             | Главная метрика: важная зона должна сохраняться. |
| **Out-of-ROI quality** | PSNR/SSIM/VMAF по обратной маске        | Контроль чрезмерной деградации периферии.        |
| **Latency/cost**       | encode time/frame, CPU/GPU, E2E latency | Нужно для real-time этапов.                      |
